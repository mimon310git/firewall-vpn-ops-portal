Portfolio notes

Common Ports

Review aid
22SSH administration. Scope source tightly and avoid public exposure.
53DNS. UDP is common for queries, TCP is used for larger answers and zone transfers.
80 / 443HTTP and HTTPS application access. HTTPS is preferred for production services.
445SMB file sharing. Treat as sensitive and keep internal-only.
3389RDP admin access. Requires strict source scope and approval.
500 / 4500IPsec IKE and NAT traversal for VPN tunnels.
1194Common OpenVPN listener, often UDP.
51820Common WireGuard listener, usually UDP.

Rule Review

Least privilege

Source should be the smallest realistic network, host, or VPN pool.

Destination should point to the exact server or subnet that owns the service.

Protocol and port should match the real application, not a broad placeholder.

Reason should include business purpose, owner, expected traffic, and review context.

Rollback should be possible by removing or disabling the generated rule.

Risk Model

Scoring logic

Low risk scoped internal access with a specific source, destination, and service.

Medium risk admin services, management networks, SMB, SSH, or RDP.

High risk broad sources like 0.0.0.0/0, broad destinations, or exposed DMZ flows.

Rejected means the request stays in history but no longer blocks duplicate review decisions.

Change Lifecycle

Ops flow
1. RequestRequester defines source, destination, service, owner, and reason.
2. ReviewApprover checks risk, scope, duplicate rules, and business need.
3. Approve / RejectApproved rules can be prepared for deployment; rejected rules stay in audit.
4. DeployOperator marks reviewed changes as deployed. Commands remain manual.
5. VerifyHealth checks and lab checks confirm expected connectivity state.
